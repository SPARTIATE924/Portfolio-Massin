## Architecture

### Static HTML Structure
- The website consists of static HTML files exported from Wix
- No build process, backend, or JavaScript framework
- Files use Wix's component-based class naming system (e.g., `comp-miykkdjp`, `wixui-*`)
- All HTML files share a common structure: header, main content, footer, and hamburger menu overlay

### Common Page Structure

All pages follow this hierarchy:
```
<div id="site-root">
  <div id="masterPage">
    <header id="SITE_HEADER">
      - Logo/name (ALIOUCHE Massin)
      - Navigation buttons (Alternance, Compétences, Projets, Veille Technologique, CV)
      - Hamburger menu button
    </header>

    <main id="PAGES_CONTAINER">
      - Page-specific content in sections
      - Column-based layouts using wixui-column-strip

    </footer>

  </div>
</div>
```

## Development Notes

### Working with HTML Files

1. **Wix Component IDs**: Component IDs are auto-generated by Wix (e.g., `comp-miykkdjp`). These IDs are consistent across the site but have no semantic meaning.

2. **CSS Classes**: The site uses Wix's utility class system:
   - Layout: `wixui-column-strip`, `wixui-section`, `mesh-layout`
   - Typography: `font_0` through `font_8`, `wixui-rich-text`
   - Components: `wixui-button`, `wixui-image`, `wixui-vertical-menu`

3. **Inline Styles**: Font sizes and specific styling are often applied via inline `style` attributes

4. **Images**: All images are hosted on Wix CDN (`static.wixstatic.com`) with responsive srcsets

### Content Updates

When updating content:
- Text content is within `wixui-rich-text` divs containing semantic HTML (`<h1>`, `<h2>`, `<p>`, `<ul>`, etc.)
- Maintain the nested `<span>` structure for styling preservation
- Links use `<a>` tags with `target="_self"` or `target="_blank"`
- Keep `class="wixui-rich-text__text"` on text elements

### No Build System

This is a static site with no compilation or build steps:
- No package.json or dependencies
- No CSS/JS bundling
- Files can be directly opened in a browser or served via any static web server

### Viewing the Site Locally

To view the site:
```bash
# Using Python
python3 -m http.server 8000

# Using Node.js
npx serve .

# Then open http://localhost:8000 in a browser
```

Note: Some features may not work locally due to hardcoded Wix URLs and CDN dependencies.
